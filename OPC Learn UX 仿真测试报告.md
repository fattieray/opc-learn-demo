# OPC Learn 前端用户体验仿真测试报告

**基于核心用户画像的全链路访问流分析与改进建议**

2026年4月

---

## 第一章：核心用户画像定义

### 1.1 画像A — 转行求职者（小林）

- **身份**：大三学生 / 转行小白
- **目标**：2小时内学会一项技能，找到实习岗位（4k-6k/月）
- **痛点**：零基础、不知道学什么、缺乏方向感
- **核心诉求**：快速上手 → 拿到认证 → 找到工作
- **典型路径**：首页 → 技能测评 → 课程中心 → 课程详情 → 加入小组 → 学习 → 互评 → 获得认证 → 个人中心

### 1.2 画像B — 进阶运营人（思琪）

- **身份**：已有1-2年经验的运营从业者
- **目标**：系统提升能力，晋升主管（10k-20k/月）
- **痛点**：能力碎片化、缺乏体系、不知道差距在哪
- **核心诉求**：体系化学习 → 管理能力 → OPC认证 → 晋升加薪
- **典型路径**：课程中心 → 体系课 → 小组学习 → 完成课程 → 认证中心 → OPC认证

### 1.3 画像C — 行业专家（林雪）

- **身份**：8年经验的资深运营人
- **目标**：知识变现，建立个人品牌
- **痛点**：有经验但不知道怎么教、缺乏教学工具
- **核心诉求**：成为认证教练 → 评审收费 → 创建付费课程
- **典型路径**：个人中心 → 专家中心 → 申请入驻 → 创建课程

## 第二章：全链路访问流仿真

### 2.1 画像A仿真流：转行求职者「小林」的首访体验

#### 步骤1：进入首页（/）

**体验描述**：小林通过朋友分享链接进入首页。看到「以实战场景驱动学习」的Hero标题，下方三个引擎卡片（微技能课入行起点、社交驱动学习、奖学金经济），以及用户旅程5步流程。

**发现的问题：**

- **P1（高）**：首页没有「登录/注册」入口，用户无法建立身份，所有个性化功能（我的学习状态、我的小组、我的认证）都无法生效
- **P2（中）**：用户旅程5步流程是静态展示，没有交互性，用户无法点击进入对应步骤
- **P3（中）**：统计数据（21个实训场景、3个行业赛道、4个学习路径）缺乏上下文，新用户不理解这些数字意味着什么
- **P4（低）**：CTA按钮「技能测评·找到你的起点」和「开始体验」并列，选择困难

#### 步骤2：技能测评（/assessment）

**体验描述**：小林点击「技能测评」进入测评页。看到介绍页，点击「开始测评」后回答5道选择题，最终得到推荐方向「零售电商」和推荐课程。

**发现的问题：**

- **P1（高）**：测评结果页面没有「注册/登录」拦截，用户看完结果后直接跳到课程详情，但没有账户体系无法保存进度
- **P2（中）**：测评只有3个行业方向（零售/文旅/制造），但用户的兴趣可能是「短视频运营」「私域运营」等技能维度，测评维度与用户心智不匹配
- **P3（低）**：测评结果页的「技能成长」进度条是写死的（+40%/+50%/+60%/+70%），不是基于用户实际测评答案动态生成的

#### 步骤3：课程中心（/courses）

**体验描述**：小林从测评结果页点击「立即开始学习」进入课程中心。看到热门微技能课推荐、体系课推荐、学习→认证→就业路径、专家课程、筛选器、课程列表。

**发现的问题：**

- **P1（高）**：页面信息密度极高，一屏内堆叠了测评入口、热门推荐、体系课推荐、认证路径、专家课程、筛选器、课程列表共7个区块，新用户容易信息过载
- **P2（高）**：筛选器（行业+课程类型）放在页面中下部，位于大量推荐内容之后，但筛选是用户找课的核心操作，应提升优先级
- **P3（中）**：课程卡片缺少视觉差异化——所有微技能课卡片结构完全相同，没有「最热门」「最适合新手」「完课率最高」等决策辅助标签
- **P4（中）**：专家课程区域与普通课程区域视觉风格接近但信息结构不同，用户可能混淆「平台课程」和「专家课程」的区别
- **P5（低）**：课程列表没有「继续学习」状态标识，已加入小组的课程和未开始的课程看起来一样

#### 步骤4：课程详情（/courses/r-micro-01）

**体验描述**：小林点击「小红书种草文案速成」进入课程详情。看到课程信息卡、课程概览、职业路径卡、四个Tab（课程内容/讨论/奖学金/小组）。

**发现的问题：**

- **P1（高）**：课程内容Tab中的模块展开/折叠交互没有动画过渡，用户点击后内容突然出现/消失，体验生硬
- **P2（高）**：学习环节的「开始」和「继续」按钮没有实际功能——点击后无任何响应，这是核心学习动作的断点
- **P3（中）**：职业路径卡片展示了「电商内容运营实习生 4k-6k/月」，但用户无法点击查看真实招聘信息（链接指向zhaopin.com首页而非具体职位）
- **P4（中）**：讨论Tab的帖子输入框「分享你的想法或问题...」是纯展示，无法实际输入和发送
- **P5（中）**：奖学金Tab的信息非常详细（资金构成、分配规则、反作弊机制、投入回报规则），但对新用户来说信息量过大，缺少「一句话总结」
- **P6（低）**：资源文档的展开/折叠需要用户逐个点击，没有「全部展开」选项

#### 步骤5：学习圈（/circles）

**体验描述**：小林从底部导航进入学习圈。看到学习动态Feed（我的学习状态、本周认可、互助问答、我帮助过的同学、最近动态）和学习小组列表两个子Tab。

**发现的问题：**

- **P1（高）**：学习动态Feed中「我的学习状态」卡片显示当前正在学习「小红书种草文案速成」，但小林作为新用户还没有加入任何小组，这里的数据与用户实际状态不一致（Mock数据写死了currentUser为小林且已加入小组）
- **P2（中）**：互助问答板块的「我来答」按钮没有实际功能
- **P3（中）**：Feed流的动态类型（提交作业、获得认可、完成课程等）没有按时间排序的视觉层次区分
- **P4（低）**：学习小组列表中，「我的」标签和小组状态（招募中/学习中/互评中/已完成）同时存在，信息冗余

#### 步骤6：个人中心（/profile）

**体验描述**：小林点击底部导航「我的」进入个人中心。看到头像、职业成长轨迹、能力成长、影响力、成长里程碑、最近成长、统计数据、荣誉进阶、学习路径、成为专家CTA、快捷链接列表。

**发现的问题：**

- **P1（高）**：页面极长（约10屏），信息密度过高。核心信息（能力成长、学习进度）和低频信息（成为专家CTA、荣誉记录、奖学金记录）混在一起
- **P2（高）**：能力成长的技能分数（文案写作20、用户洞察15）是静态Mock数据，不会随学习实际变化，用户无法感知进步
- **P3（中）**：职业成长轨迹卡片中「当前阶段→下一阶段→职业目标」的三段式展示，对新手来说「体系课」和「运营主管」太遥远，缺乏短期可达的里程碑
- **P4（中）**：统计数据（完成课程0、奖学金¥0、荣誉积分45、学习小组0）对新手来说是负面体验——全是零，没有正向激励
- **P5（低）**：快捷链接列表中有6个入口（认证中心、专家中心、我的收藏、路径详情、荣誉记录、奖学金记录），但「荣誉记录」和「奖学金记录」点击后无对应页面

### 2.2 画像B仿真流：进阶运营人「思琪」的进阶体验

#### 步骤1：从学习圈进入课程详情

**体验描述**：思琪在学习圈看到自己当前学习状态，点击「去互评」进入小组详情。

**发现的问题：**

- **P1（高）**：小组详情的「作业」Tab展示了成员评分矩阵和待点评队列，但「点评」按钮没有实际功能——这是互评流程的核心断点
- **P2（中）**：学习进度步骤（组建小组→学习资源→提交作业→同伴互评→教练点评→奖金分配）是线性展示，但思琪当前处于「互评中」阶段，页面没有突出显示她当前需要做什么
- **P3（中）**：奖金池Tab的「预计分配」基于当前评分计算，但评分数据是Mock写死的，用户无法感知自己的评分变化如何影响奖金

#### 步骤2：认证中心（/profile/certification）

**体验描述**：思琪从个人中心进入认证中心，看到三级认证体系（课程认证→OPC认证→工信部认证）。

**发现的问题：**

- **P1（高）**：认证中心展示了9张认证卡片（3个行业×3个级别），但只高亮了用户所在行业（零售），其他行业的认证以低透明度展示。对于想跨行业发展的用户，没有清晰的切换方式
- **P2（中）**：OPC认证卡片显示「准备中」状态和「继续准备」按钮，但点击后无任何响应
- **P3（中）**：认证可衔接岗位的链接指向zhaopin.com和liepin.com首页，不是具体职位搜索结果页
- **P4（低）**：三级认证之间的依赖关系（课程认证是OPC认证的前提，OPC认证是工信部认证的备考通道）虽然在底部有说明，但在认证卡片列表中没有视觉化展示

---

### 2.3 画像C仿真流：行业专家「林雪」的入驻体验

#### 步骤1：专家中心（/profile/expert）

**体验描述**：林雪从个人中心进入专家中心，看到入驻前提检查、收入模式、角色选择（认证教练/课程提供方）。

**发现的问题：**

- **P1（高）**：入驻前提检查显示「你正在准备OPC认证，通过后即可申请」，但林雪作为已有OPC认证的专家（Mock数据中experts数组包含coach-lin），这里的状态判断逻辑有误——代码用currentUser（小林）来判断，而非当前查看的专家身份
- **P2（中）**：收入模式说明（年费¥199 + 10%分成）放在角色选择之前，但用户还没决定要不要入驻，过早暴露价格可能降低转化
- **P3（中）**：「申请成为认证教练」按钮是disabled状态（因为currentUser小林没有OPC认证），但页面没有解释如何获得OPC认证的具体步骤
- **P4（低）**：平台专家列表展示了累计收入（如林雪¥8,560），但这是Mock数据，真实场景中展示收入数据可能涉及隐私问题

## 第三章：结构性问题总结

### 3.1 信息架构问题

| 编号 | 问题 | 严重度 | 影响页面 |
|------|------|--------|----------|
| IA-1 | 缺少用户身份系统（登录/注册/个人资料编辑），所有个性化功能无法生效 | <span style="color:#CC0000">**严重**</span> | 全局 |
| IA-2 | 首页定位模糊——既是落地页又是功能入口，两种角色冲突 | <span style="color:#E67E00">**高**</span> | 首页 |

## 第四章：结构化改进建议

### 4.1 Phase 1 — 核心流程打通（优先级：P0）

#### 4.1.1 用户身份系统

- 新增登录/注册页面（支持手机号+验证码，降低注册门槛）
- 新增用户引导流程：注册后→技能测评→推荐课程→加入小组
- 在首页顶部增加登录入口（不破坏现有落地页布局）

#### 4.1.2 核心学习闭环

- 实现课程内容的学习进度追踪（标记已读/未读）
- 实现作业提交功能（文本输入+文件上传）
- 实现同伴互评功能（评分+文字反馈，最低50字）
- 实现教练点评功能
- 实现小组状态流转（招募中→学习中→互评中→已完成）

#### 4.1.3 状态管理

- 引入全局状态管理（推荐Zustand），统一管理用户状态、学习进度、小组状态
- 确保Mock数据与用户实际状态一致

### 4.2 Phase 2 — 信息架构优化（优先级：P1）

#### 4.2.1 首页重构

- 方案：将首页拆分为「未登录落地页」和「已登录首页」
  - 未登录：保持现有落地页结构，强化CTA
  - 已登录：替换为个性化Dashboard（今日学习任务、进行中的课程、小组动态、技能成长）

#### 4.2.2 课程中心信息层级重组

- 新增搜索栏（放在页面顶部，支持课程名/技能名搜索）
- 将筛选器提升到搜索栏下方
- 将推荐内容改为横向滑动卡片（减少垂直空间占用）
- 课程卡片增加决策标签：「最适合新手」「完课率最高」「最热」

#### 4.2.3 个人中心分层

- 第一屏：核心信息（头像、当前课程进度、技能成长概览、快捷操作）
- 第二屏：学习数据（完成课程、奖学金、荣誉积分、学习小组）
- 第三屏：进阶入口（认证中心、专家中心、路径详情）
- 使用Tab或折叠面板减少单屏信息量

### 4.3 Phase 3 — 体验精细化（优先级：P2）

#### 4.3.1 新手激励体系

- 个人中心增加「成就墙」——展示已解锁的成就徽章
- 首次完成课程时弹出庆祝动画
- 学习连续天数打卡机制
- 技能分数动态变化动画（学习后实时更新）

#### 4.3.2 社交学习增强

- 学习圈Feed增加「与我相关」筛选
- 互助问答增加回答采纳机制
- 小组内增加@提及功能
- 学习动态增加更多类型（获得认证、奖学金到账等）

#### 4.3.3 认证体系闭环

- 实现课程认证自动发放
- 实现OPC认证申请→作品集提交→教练审核→结果通知的完整流程
- 实现工信部认证的报名→备考→考试→结果查询流程
- 认证可衔接岗位改为跳转到具体搜索结果页（带关键词预填）

#### 4.3.4 专家入驻流程

- 修复专家身份判断逻辑（基于当前登录用户，而非Mock的currentUser）
- 优化入驻流程：OPC认证→申请→审核→入驻成功
- 收入模式说明移到角色选择之后（先让用户感兴趣，再谈价格）
- 新增课程创建工具（大纲编辑器、资源上传、作业设置）

### 4.4 Phase 4 — 增长与商业化（优先级：P3）

#### 4.4.1 AI能力增强

- 技能测评结果页增加个性化学习路径推荐（基于测评答案动态生成）
