# 🚀 腾讯云轻量服务器部署指南

## ✅ 一键部署脚本已创建

**脚本位置**: [deploy-tencent-lighthouse.sh](./deploy-tencent-lighthouse.sh)

这个脚本会自动完成所有部署工作，你只需要购买服务器后运行一条命令！

---

## 📋 部署步骤

### 步骤 1: 购买轻量服务器（5 分钟）

1. **访问购买页面**: https://cloud.tencent.com/act/pro/lighthouse

2. **选择配置**:
   - **地域**: 上海（推荐）
   - **套餐**: 2核4G 5M带宽
   - **系统**: Ubuntu 22.04
   - **价格**: ¥99/年（新用户）

3. **完成购买**

4. **获取服务器信息**:
   - 访问: https://console.cloud.tencent.com/lighthouse/instance/index
   - 记录 **公网 IP**（例如: 43.136.xxx.xxx）
   - 设置或记录 **密码**

---

### 步骤 2: 连接服务器（1 分钟）

#### Windows 用户

使用 PowerShell 或 CMD：
```bash
ssh ubuntu@<你的公网IP>
# 输入密码
```

#### Mac/Linux 用户

打开终端：
```bash
ssh ubuntu@<你的公网IP>
# 输入密码
```

---

### 步骤 3: 运行一键部署脚本（15 分钟）

连接成功后，运行：

```bash
# 下载并运行部署脚本
curl -sSL https://raw.githubusercontent.com/fattieray/opc-learn-demo/main/deploy-tencent-lighthouse.sh | bash
```

**或者**，如果你已经克隆了代码：

```bash
# 进入项目目录
cd opc-learn-demo

# 运行脚本
bash deploy-tencent-lighthouse.sh
```

## 🔧 常用管理命令

### 查看服务状态

```bash
pm2 status
```

### 查看日志

```bash
# 查看所有日志
pm2 logs

# 查看前端日志
pm2 logs opc-frontend

# 查看后端日志
pm2 logs opc-backend
```

### 重启服务

```bash
# 重启所有服务
pm2 restart all

# 重启前端
pm2 restart opc-frontend

# 重启后端
pm2 restart opc-backend
```

### 更新代码

```bash
cd /opt/opc-learn

# 拉取最新代码
git pull

# 重新构建前端
npm run build

# 重启服务
pm2 restart all
```

## 🔒 配置 HTTPS（可选）

如果你有域名，可以配置 HTTPS：

### 1. 绑定域名

在域名控制台，添加 A 记录指向服务器 IP。

### 2. 配置 SSL 证书

```bash
# 运行 Certbot
sudo certbot --nginx -d 你的域名.com

# 自动续期
sudo certbot renew --dry-run
```

## ⚠️ 注意事项

### 数据库

- 当前使用 SQLite，数据存储在 `/opt/opc-learn/backend/opc_learn_demo.db`
- 建议定期备份数据库文件
- 如果需要更高可靠性，可以升级到 MySQL

### 安全

- 建议修改默认 SSH 端口
- 建议配置防火墙（安全组）
- 建议定期更新系统

### 性能

- 2核4G 可以支持约 1000 日活用户
- 如果用户增长，可以升级套餐

## 🆘 故障排查

### 网站无法访问

```bash
# 1. 检查 Nginx 状态
sudo systemctl status nginx

# 2. 检查服务状态
pm2 status

# 3. 查看错误日志
tail -f /var/log/nginx/error.log
tail -f /var/log/pm2/*.log
```

### 后端 API 错误

```bash
# 查看后端日志
pm2 logs opc-backend

# 重启后端
pm2 restart opc-backend
```

### 前端错误

```bash
# 查看前端日志
pm2 logs opc-frontend

# 重新构建前端
cd /opt/opc-learn
npm run build
pm2 restart opc-frontend
```
