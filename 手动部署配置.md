# 🎯 Render 手动部署配置指南

## 为什么不用 Blueprint？

Blueprint 的 render.yaml 文件中有 disk 配置，免费套餐不支持。
虽然我们已经修改了文件，但 Git 推送遇到问题。

**解决方案：手动创建 Web Service**

---

## 📝 手动创建步骤

### 1. 在 Render Dashboard

1. 访问：https://dashboard.render.com
2. 点击 **"New +"**
3. 选择 **"Web Service"**

### 2. 连接 GitHub

1. 选择 **"Connect"** GitHub 仓库
2. 找到并选择：`fattieray/opc-learn-demo`
3. 点击 **Continue**

### 3. 填写基本信息

| 字段 | 填写内容 |
|------|----------|
| **Name** | `opc-learn-backend` |
| **Region** | `Oregon` |
| **Branch** | `main` |
| **Root Directory** | `backend` ⚠️ 重要！ |
| **Runtime** | `Python 3` |

### 4. 配置构建和启动

| 字段 | 填写内容 |
|------|----------|
| **Build Command** | `pip install -r requirements.txt` |
| **Start Command** | `uvicorn app.main:app --host 0.0.0.0 --port $PORT` |

### 5. 添加环境变量

点击 **"Add Environment Variable"**，逐个添加：

#### 第 1 个：
```
Key: DATABASE_URL
Value: sqlite+aiosqlite:///./opc_learn_demo.db
```

#### 第 2 个：
```
Key: SECRET_KEY
Value: (点击右侧的 "Generate" 按钮自动生成)
```

#### 第 3 个：
```
Key: CORS_ORIGINS
Value: ["https://opc-learn-demo.vercel.app", "http://localhost:3000"]
```

#### 第 4 个：
```
Key: ACCESS_TOKEN_EXPIRE_MINUTES
Value: 10080
```

### 6. 不要添加 Disk

⚠️ **重要：免费套餐不支持 Disk，不要添加！**

### 7. 创建服务

滚动到页面底部，点击 **"Create Web Service"**

---

## ⏳ 部署过程

创建后，Render 会：
1. 克隆 GitHub 仓库
2. 进入 backend 目录
3. 安装依赖（约 2-3 分钟）
4. 启动应用
5. 分配域名

**总时间：约 5 分钟**

---

```

无 Disk 配置
```
