# 阿里云一体化部署方案

## 📋 方案概述

将 OPC Learning Hub 的前端（Next.js）和后端（FastAPI）一体化部署到阿里云，提供 3 种方案供选择。

---

## 🎯 方案对比

| 方案 | 产品 | 适合场景 | 成本 | 复杂度 | 推荐指数 |
|------|------|----------|------|--------|----------|
| **方案 A** | ECS 云服务器 | 完全控制、灵活配置 | ¥50-200/月 | 中等 | ⭐⭐⭐⭐ |
| **方案 B** | SAE Serverless | 自动扩缩容、免运维 | ¥30-150/月 | 低 | ⭐⭐⭐⭐⭐ |
| **方案 C** | FC 函数计算 | 按需付费、极致弹性 | ¥10-100/月 | 中等 | ⭐⭐⭐ |

---

## 🚀 方案 A：ECS 云服务器（推荐初学者）

### 优势
- ✅ 完全控制服务器环境
- ✅ 适合学习和开发
- ✅ 可以运行多个服务
- ✅ 成本可控

### 架构

```
┌─────────────────────────────────┐
│         ECS 服务器               │
│  (Ubuntu 22.04, 2C4G)          │
│                                 │
│  ┌────────────┐  ┌──────────┐  │
│  │   Nginx    │  │          │  │
│  │  反向代理   │─▶│ Next.js  │  │
│  │  :80/:443  │  │ :3000    │  │
│  └────────────┘  └──────────┘  │
│        │                       │
│        │         ┌──────────┐  │
│        └────────▶│ FastAPI  │  │
│                  │ :9000    │  │
│                  └──────────┘  │
│                       │        │
│                  ┌──────────┐  │
│                  │ SQLite   │  │
│                  │ Database │  │
│                  └──────────┘  │
└─────────────────────────────────┘
```

### 部署步骤

#### 1. 购买 ECS 实例

```bash
# 推荐配置
- 地域：华北2（北京）或根据你的用户位置选择
- 实例规格：ecs.t6-c1m2.large (2核4G)
- 操作系统：Ubuntu 22.04 LTS
- 系统盘：40GB SSD
- 带宽：按量付费 5Mbps
- 价格：约 ¥80-120/月
```

#### 2. 配置安全组

```bash
# 开放端口
- 22 (SSH)
- 80 (HTTP)
- 443 (HTTPS)
- 3000 (Next.js, 可选)
- 9000 (FastAPI, 可选)
```

#### 3. 连接服务器

```bash
ssh root@<你的ECS公网IP>
```

#### 4. 安装基础环境

```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt install -y nodejs

# 安装 Python 3.10+
apt install -y python3 python3-pip python3-venv

# 安装 Nginx
apt install -y nginx

# 安装 Git
apt install -y git

# 安装 PM2 (进程管理)
npm install -g pm2
```

#### 5. 部署后端 (FastAPI)

```bash
# 创建应用目录
mkdir -p /opt/opc-learn
cd /opt/opc-learn

# 克隆代码
git clone https://github.com/fattieray/opc-learn-demo.git
cd opc-learn-demo/backend

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 创建环境变量
cat > .env << EOF
DATABASE_URL=sqlite+aiosqlite:///./opc_learn_demo.db
SECRET_KEY=your-secret-key-here
CORS_ORIGINS=["http://<你的域名>","https://<你的域名>"]
EOF

# 测试运行
uvicorn app.main:app --host 0.0.0.0 --port 9000
```

#### 6. 使用 PM2 管理后端进程

```bash
# 创建 PM2 配置
cat > /opt/opc-learn/opc-learn-demo/backend/ecosystem.config.js << EOF
module.exports = {
  apps: [{
    name: 'opc-backend',
    script: 'venv/bin/uvicorn',
    args: 'app.main:app --host 0.0.0.0 --port 9000',
    cwd: '/opt/opc-learn/opc-learn-demo/backend',
    instances: 2,
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'production',
      DATABASE_URL: 'sqlite+aiosqlite:///./opc_learn_demo.db',
      SECRET_KEY: 'your-secret-key-here'
    },
    error_file: '/var/log/pm2/opc-backend-error.log',
    out_file: '/var/log/pm2/opc-backend-out.log',
    merge_logs: true,
    autorestart: true,
    max_restarts: 10,
    min_uptime: '10s'
  }]
};
EOF

# 创建日志目录
mkdir -p /var/log/pm2

# 启动服务
cd /opt/opc-learn/opc-learn-demo/backend
pm2 start ecosystem.config.js

# 设置开机自启
pm2 startup
pm2 save

# 查看状态
pm2 status
pm2 logs opc-backend
```

#### 7. 部署前端 (Next.js)

```bash
cd /opt/opc-learn/opc-learn-demo

# 创建环境变量
cat > .env.production << EOF
NEXT_PUBLIC_API_URL=http://localhost:9000/api/v1
EOF

# 安装依赖并构建
npm install
npm run build

# 使用 PM2 启动
pm2 start npm --name "opc-frontend" -- start

# 保存配置
pm2 save
```

#### 8. 配置 Nginx 反向代理

```bash
# 创建 Nginx 配置
cat > /etc/nginx/sites-available/opc-learn << 'EOF'
server {
    listen 80;
    server_name <你的域名或IP>;

    # 前端 Next.js
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # 后端 FastAPI
    location /api/ {
        proxy_pass http://localhost:9000/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # CORS headers
        add_header Access-Control-Allow-Origin *;
        add_header Access-Control-Allow-Methods 'GET, POST, PUT, DELETE, OPTIONS';
        add_header Access-Control-Allow-Headers 'DNT,X-Mx-ReqToken,Keep-Alive,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type,Authorization';
        
        if ($request_method = 'OPTIONS') {
            return 204;
        }
    }

    # 静态资源缓存
    location /_next/static/ {
        proxy_pass http://localhost:3000;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
EOF

# 启用配置
ln -s /etc/nginx/sites-available/opc-learn /etc/nginx/sites-enabled/
rm /etc/nginx/sites-enabled/default

# 测试配置
nginx -t

# 重启 Nginx
systemctl restart nginx
systemctl enable nginx
```

#### 9. 配置 HTTPS (Let's Encrypt)

```bash
# 安装 Certbot
apt install -y certbot python3-certbot-nginx

# 获取 SSL 证书
certbot --nginx -d <你的域名>

# 自动续期
certbot renew --dry-run
```

#### 10. 配置数据库备份

```bash
# 创建备份脚本
cat > /opt/opc-learn/backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/opt/opc-learn/backups"
DATE=$(date +%Y%m%d_%H%M%S)
DB_FILE="/opt/opc-learn/opc-learn-demo/backend/opc_learn_demo.db"

mkdir -p $BACKUP_DIR
cp $DB_FILE $BACKUP_DIR/opc_learn_$DATE.db

# 保留最近 30 天的备份
find $BACKUP_DIR -name "*.db" -mtime +30 -delete

echo "Backup completed: $DATE"
EOF

chmod +x /opt/opc-learn/backup.sh

# 添加定时任务 (每天凌晨 2 点备份)
crontab -e
# 添加: 0 2 * * * /opt/opc-learn/backup.sh >> /var/log/opc-backup.log 2>&1
```

## ☁️ 方案 B：SAE Serverless（推荐生产环境）

### 优势
- ✅ 自动扩缩容
- ✅ 按实际使用付费
- ✅ 免运维
- ✅ 内置监控和日志

### 架构

```
┌──────────────────────────────────────┐
│          阿里云 SAE                   │
│                                      │
│  ┌────────────┐    ┌────────────┐   │
│  │  前端应用   │    │  后端应用   │   │
│  │  Next.js   │───▶│  FastAPI   │   │
│  │  :8080     │    │  :8080     │   │
│  └────────────┘    └────────────┘   │
│                                      │
│  ┌────────────┐                     │
│  │   NAS/OSS  │ (持久化存储)        │
│  │  Database  │                     │
│  └────────────┘                     │
└──────────────────────────────────────┘
        │
        ▼
┌──────────────┐
│  SLB 负载均衡 │
│  域名绑定     │
└──────────────┘
```

### 部署步骤

#### 1. 准备工作

```bash
# 安装阿里云 CLI
pip install aliyun-cli

# 配置凭证
aliyun configure
# 输入 AccessKey ID 和 AccessKey Secret
```

#### 2. 容器化后端

```bash
# 创建 Dockerfile (backend/Dockerfile)
cat > backend/Dockerfile << 'EOF'
FROM python:3.10-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8080

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8080"]
EOF

# 构建镜像
cd backend
docker build -t registry.cn-beijing.aliyuncs.com/<你的命名空间>/opc-backend:latest .

# 登录阿里云容器镜像服务
docker login registry.cn-beijing.aliyuncs.com

# 推送镜像
docker push registry.cn-beijing.aliyuncs.com/<你的命名空间>/opc-backend:latest
```

#### 3. 容器化前端

```bash
# 创建 Dockerfile (根目录 Dockerfile)
cat > Dockerfile << 'EOF'
FROM node:18-alpine AS builder

WORKDIR /app
COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

FROM node:18-alpine AS runner
WORKDIR /app

COPY --from=builder /app/package*.json ./
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public

ENV NEXT_PUBLIC_API_URL=http://<后端SAE地址>/api/v1

EXPOSE 8080

CMD ["node", "server.js"]
EOF

# 构建并推送
docker build -t registry.cn-beijing.aliyuncs.com/<你的命名空间>/opc-frontend:latest .
docker push registry.cn-beijing.aliyuncs.com/<你的命名空间>/opc-frontend:latest
```

#### 4. 创建 SAE 应用

```bash
# 通过阿里云控制台创建
1. 登录 SAE 控制台
2. 创建应用 -> 使用镜像创建
3. 配置:
   - 应用名称: opc-backend
   - 镜像: registry.cn-beijing.aliyuncs.com/<你的命名空间>/opc-backend:latest
   - 端口: 8080
   - 实例数: 2
   - 规格: 1核2G
   - 环境变量:
     DATABASE_URL=sqlite+aiosqlite:///./data/opc_learn.db
     SECRET_KEY=your-secret-key

4. 同样创建 opc-frontend 应用
```

#### 5. 配置 NAS 持久化存储

```bash
# 在 SAE 控制台
1. 应用设置 -> 存储配置
2. 添加 NAS 挂载点
3. 挂载路径: /data
4. 用于存储 SQLite 数据库文件
```

#### 6. 配置 SLB 负载均衡

```bash
# 在 SLB 控制台
1. 创建负载均衡实例
2. 添加前端监听器 (HTTP:80, HTTPS:443)
3. 添加后端监听器 (/api/* 路由到后端应用)
4. 绑定域名
```

## ⚡ 方案 C：函数计算 FC（适合低流量场景）

### 优势
- ✅ 按需付费，成本极低
- ✅ 自动扩缩容到 0
- ✅ 无需管理服务器

### 架构

```
┌─────────────────────────────────┐
│         阿里云函数计算 FC        │
│                                 │
│  ┌──────────┐  ┌────────────┐  │
│  │ 前端函数  │  │  后端函数   │  │
│  │ Next.js  │  │  FastAPI   │  │
│  │ HTTP触发器│  │  HTTP触发器 │  │
│  └──────────┘  └────────────┘  │
│                                 │
│  ┌──────────┐                  │
│  │   RDS    │ (MySQL/PostgreSQL)
│  │ Database │                  │
│  └──────────┘                  │
└─────────────────────────────────┘
```

### 部署步骤

#### 1. 安装 Serverless Devs

```bash
npm install -g @serverless-devs/s
```

#### 2. 配置凭证

```bash
s config add
# 选择阿里云
# 输入 AccessKey ID 和 Secret
```

#### 3. 创建部署配置

```yaml
# s.yaml
edition: 3.0.0
name: opc-learn-app
access: default

vars:
  region: cn-beijing

resources:
  backend:
    component: fc3
    props:
      region: ${vars.region}
      functionName: opc-backend
      description: OPC Learn Backend API
      runtime: custom.debian10
      codeUri: ./backend
      customRuntimeConfig:
        command:
          - uvicorn
          - app.main:app
          - --host
          - '0.0.0.0'
          - --port
          - '9000'
        port: 9000
      environmentVariables:
        DATABASE_URL: sqlite+aiosqlite:///tmp/opc_learn.db
        SECRET_KEY: your-secret-key
      triggers:
        - triggerName: httpTrigger
          triggerType: http
          triggerConfig:
            authType: anonymous
            methods:
              - GET
              - POST
              - PUT
              - DELETE

  frontend:
    component: fc3
    props:
      region: ${vars.region}
      functionName: opc-frontend
      description: OPC Learn Frontend
      runtime: custom.debian10
      codeUri: ./
      customRuntimeConfig:
        command:
          - npm
          - run
          - start
        port: 3000
      environmentVariables:
        NEXT_PUBLIC_API_URL: ${resources.backend.props.url}
      triggers:
        - triggerName: httpTrigger
          triggerType: http
          triggerConfig:
            authType: anonymous
            methods:
              - GET
```

#### 4. 部署

```bash
# 部署所有资源
s deploy

# 查看部署结果
s info
```

## 🎯 推荐方案

### 学习/开发环境 → 方案 A (ECS)
- 完全控制，方便调试
- 成本低，适合长期使用
- 可以同时运行多个项目

### 生产环境/正式上线 → 方案 B (SAE)
- 自动扩缩容，应对流量波动
- 内置监控和日志
- 免运维，专注业务开发

### 低流量/测试环境 → 方案 C (FC)
- 成本最低
- 按实际使用付费
- 适合 Demo 展示
