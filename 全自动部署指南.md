## ✅ 部署后验证

### 1. 初始化数据库

```bash
# 替换为你的实际 Render URL
RENDER_URL="https://opc-learn-backend.onrender.com"

# 初始化数据库
curl -X POST $RENDER_URL/api/v1/seed/

# 应该返回：
# {"status": "success", "message": "Database seeded successfully"}
```

### 2. 测试 API

```bash
# 测试课程列表
curl $RENDER_URL/api/v1/courses/

# 测试用户注册
curl -X POST $RENDER_URL/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"phone": "13800138000", "nickname": "测试用户"}'
```

### 3. 连接前端

```bash
# 配置 Vercel 环境变量
cd opc-learn-demo
vercel env add NEXT_PUBLIC_API_URL production

# 输入你的 Render URL（加 /api/v1）：
# https://opc-learn-backend.onrender.com/api/v1

# 重新部署前端
vercel --prod
```

## 🆘 故障排查

### 问题 1: Render 找不到 render.yaml

**解决：**
- 确保在 Blueprint 中选择正确的仓库
- 检查 render.yaml 在根目录

### 问题 2: 部署失败

**查看日志：**
- Dashboard → 服务 → **Logs** 标签

**常见错误：**
- `requirements.txt not found` → Root Directory 应为 `backend`
- 依赖安装失败 → 检查 requirements.txt 格式
- 启动失败 → 检查 Procfile 格式

### 问题 3: CORS 错误

**确保 CORS_ORIGINS 包含前端 URL：**
```json
["https://opc-learn-demo.vercel.app", "http://localhost:3000"]
```

### 问题 4: 数据丢失

**检查持久化存储：**
- Dashboard → 服务 → **Disks** 标签
- 确保 Disk 已挂载到 `/opt/render/project/src/data`
